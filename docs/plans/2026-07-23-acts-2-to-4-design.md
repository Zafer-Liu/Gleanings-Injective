# 《拾遗》幕二至幕四可玩闭环设计

> 确认日期：2026-07-23  
> 状态：用户逐段确认  
> 范围：幕二《冬酿》、幕三《月子酒》、幕四《归》的纯前端可玩版本

## 1. 目标与边界

本阶段把第一章从“第一幕可玩”扩展为能够从开坛连续体验到最终徽章的完整 RPG 叙事闭环。

必须交付：

1. 幕二、幕三具有大于视口的可探索地图、碰撞、遮挡、NPC、任务与物件状态变化；
2. 记忆幕切换玩家身份：幕二控制少女太婆，幕三控制中年太婆；
3. 幕四回到林怡视角，回收前三幕选择并生成本地双语文化酒签；
4. 冬酿曲印、青花酒盏和“一坛回声”徽章进入章节收藏；
5. 章节级存档、安全检查点和第一幕 v1 存档迁移；
6. 从第一幕开坛到幕四结算的浏览器端完整路径；
7. 剧情结算后衔接一支约 80 秒、可跳过和重播的福建老酒与黄酒文化短片。

本阶段不接入真实 LLM、钱包、voucher 服务或链上交易。AI 结果使用受控本地模板，铸造使用明确标注的演示状态；二者不得伪装成真实外部服务。

## 2. 架构

采用“共享章节状态 + 每幕独立场景”的混合结构。

- 共享层负责章节存档、选择路径、信物、文化卡、任务 HUD、对白、选择面板、幕间转场和章节结算；
- 每幕拥有独立地图内容、任务 reducer、交互物和演出逻辑；
- 通用地图几何、脚底碰撞、前景遮挡与玩家移动沿用第一幕已验证实现；
- 不在本阶段制作通用关卡编辑器，也不把三幕全部硬编码进一个大型场景。

章节状态以 `ChapterOneSaveV2` 为唯一持久化入口。第一幕原有 reducer 可以继续承担第一幕内部推进，但保存时映射进章节状态。载入旧 `gleanings.act1.save.v1` 时迁移为 v2，并保留感官选择、背包、完成状态和最近玩家位置。

## 3. 幕二《冬酿》

### 3.1 场景

- 地图：`44×30` tiles，原生 `1408×960`；
- 视口：`640×360`；
- 分区：入场区、阿凤师工位、曲房、陶坛区、拌曲台、下缸区、封坛区；
- 视觉：冬霜灰、陶褐、红曲朱；无渐变、环境光或伪 3D；
- 角色：玩家控制少女太婆，阿凤师为任务 NPC。

现有 `40×28` 酒坊地图作为构图与材质基础，扩展边缘与分区，不做简单拉伸。玩家和 NPC 不烘焙进背景。

### 3.2 主流程

```text
记忆醒来
  → 与阿凤师交谈
  → 检视红曲盘
  → 检视陶坛
  → 取得红曲样品
  → 霜降：拌红曲
  → 立冬：下缸
  → 冬至：封坛
  → 向阿凤师追问
  → 获得冬酿曲印
  → 进入幕三
```

三个工序没有失败判定。顺序不正确时只给出当前节气提示，不消耗物品。每完成一步，工位贴图、任务文本、旁白和进度标记同步变化。

阿凤师在介绍酒曲时轻量埋下地域线索：

> 黄酒各地都有自己的脾气。我们福建这坛酒，认的是红曲。

本幕只建立“福建老酒—红曲—黄酒”的关系，不在角色对白中展开黄酒通史，避免打断任务节奏。

核心选择：

- `ask_hongqu`：解锁《古田红曲》，普通曲印；
- `ask_winter`：解锁《冬酿与节气》，稀有曲印；
- `ask_future`：解锁隐藏卡；若第一幕选择“坛身的凉”，生成传世“留白印”，否则为稀有。

## 4. 幕三《月子酒》

### 4.1 场景

- 地图：`36×26` tiles，原生 `1152×832`；
- 分区：灶间、储物间、酒坛架、餐桌区、产妇休息区；
- 角色：玩家控制中年太婆，阿珍与家人为 NPC；
- 关键物件：碗、面线、老酒、灶台、成品面线、青花酒盏；
- 灶火只作为明确可见的实体阻挡，不造成伤害。

### 4.2 主流程

```text
多年后醒来
  → 与家人交谈
  → 取得碗
  → 取得面线
  → 舀取老酒
  → 回到灶台烹制
  → 盛好老酒面线
  → 交给阿珍
  → 回应这份人情
  → 获得青花酒盏
  → 返回现代
```

三件材料允许任意收集顺序。背包和任务 HUD 显示 `0/3` 到 `3/3`。烹制完成后，灶台、餐桌、阿珍和家人的位置改变，制造一次明确的场景状态回收。

核心选择：

- `receive_warmth`：铭文“暖”，结局基调温暖；
- `ask_burden`：铭文“承”，结局基调敬意；
- `watch_hands`：铭文“念”，结局基调怅惘。

## 5. 幕四《归》

幕四复用 `30×20` 公寓地图，不扩大地图。场景使用开坛后物件状态：酒坛已开、任务箭头移至米娅或酒签、纸箱任务关闭。

主流程：

```text
回到公寓
  → 与米娅交谈
  → 选择如何用英文解释老酒
  → 展开选择册页
  → 生成本地双语文化酒签
  → 接受或重试一次
  → 演示铸造
  → 获得“一坛回声”
  → 家族剧情完成
  → 进入黄酒文化短片
  → 章节完成页
```

米娅在酒签生成前提出连接问题：

> 这是你们家的酒，还是福建人都会酿的酒？

林怡回应：

> 它从我们家出发，但背后是一条更长的黄酒传统。

这两句只负责把镜头从家族记忆推向地域与历史。演示铸造和“一坛回声”仍是情感结局；文化短片在结局之后独立播放，不用科普覆盖角色关系的收束。

文化酒签固定包含：

- 中文名与英文名；
- 一句海外读者可独立理解的英文说明；
- 明确标记为“文化事实”的只读短注；
- 明确标记为“创意表达”的玩家旅程短文；
- 第一幕感官、幕二追问、幕三铭文和幕四表达角度；
- 本地模板来源标记。

重试只允许一次，并在同一选择集合内切换表达模板，不改变文化事实。演示铸造不请求钱包，只生成本地 token 占位编号与 `pathHash` 展示。

## 6. 片尾文化短片

### 6.1 定位与播放规则

- 片名：《从一坛福建老酒，到中国黄酒》；
- 时长：约 80 秒；
- 入口文案：“故事已经结束，酒的故事还没有。”；
- 在“一坛回声”结算完成后播放，不影响徽章获得与章节存档；
- 支持暂停、跳过、字幕开关和结算页重播；
- 跳过视频后仍进入同一章节完成页；
- 首版使用本地视频文件或可由静帧、字幕与旁白组成的本地时间轴，不依赖网络流媒体。

### 6.2 叙事结构

```text
00–10 秒  从太婆留下的酒坛与福建家庭记忆起笔
10–24 秒  介绍黄酒是中国代表性的谷物酿造酒
24–38 秒  贾湖早期混合发酵饮料遗存
38–55 秒  酒曲与边糖化边发酵的技术特点
55–68 秒  宋元时期酿造、澄清与储存技术发展
68–80 秒  回到福建红曲黄酒与代际传承
```

最后一帧文案：

> 酒会饮尽，酿酒的人与他们的故事，不该消失。

### 6.3 审定旁白

> 黄酒，是中国代表性的谷物酿造酒，也是世界上历史最悠久的酿造酒之一。
>
> 约九千年前，贾湖先民已经开始制作含有稻米、蜂蜜与果实的发酵饮料。它并不是今天的黄酒，却让我们看见了中国古老的酿造传统。
>
> 后来，酒曲让谷物的糖化与酒精发酵在同一过程中推进，形成了中国谷物酿造极具辨识度的技术道路。经过漫长发展，黄酒的酿造、澄清和储存方式日渐成熟。
>
> 在福建，红曲为黄酒赋予了独特的颜色、滋味与地方记忆。一坛福建老酒，装着的不只是酒，也是人与故乡之间一代又一代的联系。

片中如出现分类文字，使用“黄酒常与啤酒、葡萄酒并列为世界三大酿造酒”，不使用“世界三大古酒”或“唯一源自中国”等绝对表述。酒精度不作为核心旁白；若信息卡必须展示，采用现行生产许可口径“酒精度不低于 8%、低于 24%”。

贾湖表述必须明确其为稻米、蜂蜜与果实的早期混合发酵饮料遗存，不称作“最早的黄酒”。宋元表述使用“技术继续发展、形态逐渐稳定”，不使用单线式的“由浊转清”。

事实核验依据：

- [PNAS：Fermented beverages of pre- and proto-historic China](https://pubmed.ncbi.nlm.nih.gov/15590771/)
- [国家市场监督管理总局：酒类生产许可审查细则（黄酒等）](https://www.samr.gov.cn/spscs/tzgg/art/2023/art_c7f0624fa4fe46daa509549b5ba85ab3.html)
- [福建省人民政府：老味道寻求科学表达](https://www.fujian.gov.cn/zwgk/ztzl/sxzygwzxsgzx/flsxkmh/202501/t20250109_6620349.htm)
- [江南大学传统酿造研究中心：黄酒的发展历程（三）](https://rctff.jiangnan.edu.cn/info/1071/1652.htm)

## 7. 素材

复用：

- 第一幕林怡、米娅、公寓、HUD、对白、选择、转场；
- 现有酒坊地图、酒坊 tileset、少女太婆风格稿、红曲盘、红曲样品、冬酿曲印。

新增：

- `44×30` 无角色酒坊完整背景；
- 少女太婆与阿凤师 `96×192` 行走表；
- 三个酒坊工位的未完成/完成状态；
- `36×26` 灶间完整背景；
- 中年太婆、阿珍 `96×192` 行走表；
- 碗、面线、老酒、灶台状态、成品面线、青花酒盏；
- 章节徽章与酒签展示所需的本地组合层；
- 文化短片所需的贾湖器皿、谷物、酒曲复式发酵、宋元酒坊、福建红曲与酒坛回环画面；
- 视频中英字幕、章节标题卡、事实来源卡和结尾题字。

全部新增位图遵守固定 24 色、最近邻缩放、硬像素边缘和透明通道 `0/255`。关键素材加载失败时显示资源 ID；非关键装饰缺失时继续运行。

文化短片延续游戏的硬边像素风，但采用横向分层卷轴、器物特写和信息图式构图，不模拟普通 RPG 行走画面。不得使用与游戏色板冲突的写实素材、柔光滤镜或现代酒类广告式镜头。

## 8. 存档与数据

章节存档至少包含：

```ts
type ChapterOneSaveV2 = {
  version: 2;
  currentAct: 1 | 2 | 3 | 4 | "complete";
  checkpoint: string;
  act1Sense: "aroma" | "hongqu_red" | "cold_clay" | null;
  act2Question: "ask_hongqu" | "ask_winter" | "ask_future" | null;
  act3Inscription: "warm" | "inherit" | "remember" | null;
  act4Explanation: "letter" | "three_generations" | null;
  inventory: string[];
  relics: string[];
  cultureCards: string[];
  playerTile: { x: number; y: number };
  labelRetryUsed: boolean;
  cultureFilmSeen: boolean;
  chapterComplete: boolean;
};
```

安全检查点：

- 幕二：取得样品、每个工序完成、获得曲印；
- 幕三：每个材料取得、烹制完成、获得酒盏；
- 幕四：完成解释选择、接受酒签、演示铸造完成；
- 片尾：首次播完或跳过文化短片。

## 9. 验收

1. 第一幕旧存档能进入幕二，不丢失感官选择；
2. 幕二、幕三地图大于视口，摄像机跟随、碰撞和遮挡正确；
3. 任务顺序、任意材料收集、选择和信物变体均有 reducer 测试；
4. 刷新可从所有安全检查点恢复；
5. 幕四能回收四次选择并生成稳定酒签；
6. 情感结局先于文化短片成立，跳过视频不会影响徽章和章节完成状态；
7. 文化短片可播放、暂停、跳过、开关字幕，并可从完成页重播；
8. 贾湖、黄酒工艺、宋元发展与福建红曲的字幕口径符合本设计的事实边界；
9. 单元测试、类型检查、生产构建和完整 Playwright 路径全部通过；
10. 主流程不依赖网络、钱包、AI、链上服务或网络视频。
