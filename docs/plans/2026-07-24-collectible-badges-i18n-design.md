# 收藏徽章、国际化与墨水屏设计

## 目标

把“剧情中的临时道具”和“每幕完成后的永久徽章”拆成两套稳定概念。两章各四幕，共八枚徽章；徽章进入收藏馆、可以选择上链，并在 296×152 墨水屏展签上保持清晰可辨。

## 数据结构

新增统一收藏目录，以稳定英文 ID 和 `i18nKey` 作为业务主键：

- `kind` 只允许 `item | badge`，不再用中文字符串驱动逻辑。
- 每项同时保存中文、英文名称、说明和来源。
- `rewardIds` 记录剧情存档中的旧奖励 ID；同一幕的分支奖励解析成同一枚徽章。
- 收藏馆根据第一章旧存档、第一章 v2 存档和第二章存档推导已解锁收藏。
- 太婆字条保留为“道具”；剧情中会被消耗的样品、食材和热面线不进入永久收藏。

## 兼容与上链

- 旧存档 ID 不迁移、不重写，避免损坏进度。
- 旧链上 `collectible_id`、`medal_id`、中文 `category` 继续可读，并归一化到新的稳定徽章 ID。
- 新铸造元数据写入 `category: badge | item`、`i18n_key`、`name_zh`、`name_en`、双语说明和双语来源。
- 收藏馆提供“全部 / 徽章 / 道具”筛选；所有已解锁徽章均显示上链入口。

## 墨水屏视觉

八枚徽章各有一张独立的黑白线稿 SVG：

- 白底为主体，黑色轮廓和少量实心识别点；
- 避免大面积深色填充和依赖灰度的细节；
- 在 132×132 展签区域内保留安全边距；
- 徽章 SVG 直接缩放合成，不再经过会导致“全黑”的全局阈值化；
- 普通彩色道具仍走灰度归一化处理。

自动测试会检查输出尺寸、每枚徽章同时包含足够白色和黑色像素、黑色覆盖率不过高，从而阻止“全黑徽章”回归。

## 验收标准

1. 完成任一幕，对应徽章会出现在收藏馆。
2. 八枚徽章名称与上表一致，最后一幕分别为“福建老酒”和“西湖龙井”。
3. 收藏馆可以按“徽章 / 道具”筛选。
4. 每枚徽章都能发起上链，元数据包含稳定类型和中英文字段。
5. 旧存档和旧链上藏品仍能显示。
6. 八枚徽章在墨水屏展签上轮廓清楚，不出现全黑图块。
